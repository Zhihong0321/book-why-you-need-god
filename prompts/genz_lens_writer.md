## 第四步：生成 Gen Z lens 内容

### Gen Z 读者是谁

**年龄：** 约 18–30 岁

**他们的世界：**
- 活在自我优化文化里（健身、冥想、心理咨询、个人成长播客）
- 对"表演性善良"有本能的识别力——知道什么是真实的，什么是人设
- 心理健康是日常语言：burnout、trauma、healing、authentic self
- 对机构（教会、政府、企业）高度不信任，但对"真实的人"开放
- 不反对"神"这个概念，但拒绝被评判、被控制、被说教
- 很多人是"高功能空心人"：简历漂亮，内心有一个无法命名的空洞

**他们深处的感受（但很少说出来）：**
- "我已经做了所有对的事，为什么还是感觉不够？"
- "我一直在修复自己，但有些东西修不好。"
- "我很累。"

### 写作规则

**必须做：**
- 从他们的**真实感受或处境**开始，不从逻辑开始
- 让他们感到**被看见**，再引出章节的核心问题
- 用当代语言：可以用"系统性问题"、"人设"、"burnout"、"自我优化"等词
- 类比优先选：心理健康、社交媒体、游戏、职场、亲密关系
- 结尾自然带入章节核心问题，不强行给结论

**绝对不能做：**
- 说教：不出现"你应该"、"你必须"
- 评判：不暗示他们的生活方式是错的
- 卖弄：不出现哲学家名字（康德、黑格尔等）
- 强行用网络用语（保持真实自然）
- 给答案太快：让问题先在读者心里停留

**长度：200–350 字**

**段落结构：**
1. **处境钩子**（2–4句）：描述一个他们可能活过的具体感受。不用问句，直接陈述。让他们认出自己。
2. **命名那个感受**（2–3句）：把那个感受说得更精确。他们知道这个感觉，但可能从未有人这样说出来。
3. **转折**（1–2句）：提出这章的核心问题——不是攻击，而是他们自己也想知道答案的问题。
4. **邀请**（1句）：不是"让我告诉你答案"，而是"我们来看看这是怎么回事"。

### 示例（COGST 章）

```
### 【年轻人视角】

你大概认识这样的人——或者你就是这样的人。成绩不错，从不惹麻烦，对朋友真心，工作认真。没有做过什么大错事。如果要列一张"好人清单"，你应该在上面。

但有时候，在睡前或者某个发呆的瞬间，你会有一种说不清楚的感觉：这些努力和积累，好像没有真正解决什么。你做了很多对的事，但那个空洞还在那里。你不知道它叫什么名字。

这一章问的问题，很多"好人"都不敢问：如果善良本身不是答案——那答案是什么？

我们来看。
```

## 第五步：写入文件

在 `## 逻辑` 层内，找到该层末尾的 `*[继续深入 →]*`。

用 Edit 工具插入，规则如下：
- `old_string`：`*[继续深入 →]*`（逻辑层末尾那一个）
- `new_string`：Gen Z lens 内容 + 原来的 `*[继续深入 →]*`

插入后结构为：

```
## 逻辑
[原有内容]

> 【直觉窗口：投影案例】
> [原有类比]

---

### 【年轻人视角】

[新写入的内容]

*[继续深入 →]*
```

写入完成后，告知用户："✅ 已写入 [文件名]，Gen Z 视角已添加至 ## 逻辑 层。"

---

## 注意

- 不得修改文件中任何其他内容
- 如果文件中 `*[继续深入 →]*` 出现多次，只操作 `## 逻辑` 层内的那一个
- 如果该章 `## 逻辑` 层已存在 `### 【年轻人视角】`，告知用户已存在，询问是否覆盖
